# palimpsestr v0.13.0 — Design Spec

**Date**: 2026-05-02
**Branch**: `feat/v0.13.0` (off `master` @ `bdf93af`)
**Target version**: `0.13.0` (minor bump, additive only, no breaking changes)
**Authors of spec**: Enzo Cocca (with Claude as design assistant)

---

## 1. Goals & scope

Three feature additions corresponding to items 1, 3 and 4 of the "Future development" section of the v0.12.0 paper (`docs/palimpsestr-paper.Rmd`, lines 389–397):

1. **Directional intrusion diagnostics** — extend `detect_intrusions()` so that each flagged find is classified as `older_than_context`, `in_context`, or `younger_than_context`, using interval overlap between the find's chronology and the (leave-one-out) chronological envelope of its stratigraphic unit.
2. **Type longevity** — new function `type_longevity()` that, for each cultural class, returns the temporal envelope of the phases in which that class has non-negligible posterior membership.
3. **Calibrated 14C adapter** — new function `chronology_from_rcarbon()` that converts a `rcarbon::CalDates` object into the `date_min`/`date_max`/`date_mid` triple used by `fit_sef()`, with three reduction methods (HPD, median+IQR, weighted mean+SD).

**Out of scope** (deferred to v0.14 or later):

- KDE-based variant of type longevity (`type_longevity_kde()`).
- Soft-constraint integration of full 14C posterior densities directly inside the EM (requires real cemetery data, expected ~June 2026).
- Lago Lucone dendrochronology case study (requires Baioni's collaboration data).

## 2. Non-goals & invariants

- **No breaking changes**. Existing user code that calls `detect_intrusions(fit)` and reads `intrusion_prob` continues to work; the only change is two new columns appended.
- **No new mandatory dependency**. `rcarbon` lives in `Suggests` and is gated by `requireNamespace()`.
- **No change to the EM algorithm** or to `fit_sef()` defaults.
- **No removal of v0.12.0 functions**.

## 3. Feature 1 — Directional intrusion diagnostics

### 3.1 Function

`detect_intrusions(object)` in `R/fit.R` — same name and signature as v0.12.0; output schema extended.

The function reads the column names for stratigraphic context, `date_min`, and `date_max` from the metadata that `fit_sef()` already stores in the returned `sef_fit` object (since v0.12.0 these are persisted in `object$call_args`/`object$context_col`/`object$date_cols` — exact slot names to be confirmed when reading `R/fit.R` during implementation; if not currently persisted, persist them as part of the same commit since the cost is one extra line in `fit_sef()`).

### 3.2 Algorithm (interval overlap, leave-one-out)

For each find `i`:

1. Read `[d_min_i, d_max_i]` from the data slot of `object$data` (columns named `date_min`/`date_max`, configurable via the same convention used by `fit_sef()`). If only one of the two is present, treat the find as point-dated at that value (i.e. `d_min_i == d_max_i`). If both are missing, mark as `NA`.
2. Group finds by stratigraphic unit (`object$context_col` if stored, else the column passed to `fit_sef()`; recovered from the fit object's metadata).
3. Build the unit envelope **excluding find `i`** (leave-one-out): `U_min = min(d_min_j) for j in unit, j != i`; `U_max = max(d_max_j) for j in unit, j != i`. If the unit has only one find (i.e. only `i`), the comparison is undefined → `direction = NA`, `chrono_gap = NA`.
4. Classify:
   - `d_max_i < U_min` → `direction = "older_than_context"`, `chrono_gap = d_max_i - U_min` (negative).
   - `d_min_i > U_max` → `direction = "younger_than_context"`, `chrono_gap = d_min_i - U_max` (positive).
   - Intervals overlap → `direction = "in_context"`, `chrono_gap = 0`.

### 3.3 Output schema

```r
data.frame(
  id              = character or original-type,
  intrusion_prob  = numeric in [0, 1],         # unchanged from v0.12.0
  direction       = factor(levels = c("older_than_context",
                                      "in_context",
                                      "younger_than_context")),
  chrono_gap      = numeric                    # years, signed
)
```

### 3.4 Edge cases & messaging

- No chronology available at all in `object$data` (neither `date_min` nor `date_max`): all rows have `direction = NA`, `chrono_gap = NA`, and exactly one `warning()` is emitted per `detect_intrusions()` call (not one per row): `"no chronological columns available; 'direction' set to NA for all rows"`. No error.
- Unit with a single find: that find's row has `NA, NA`. No warning.
- Find with missing chronology in a unit otherwise populated: same — `NA, NA`, no warning.

### 3.5 Internals

Compute the per-find direction in a private helper `.compute_direction(data, context_col, date_min_col, date_max_col)` in `R/utils.R`. Vectorised by unit using `split()` + `lapply()`; no nested loops on individual finds.

## 4. Feature 2 — Type longevity

### 4.1 Function

`type_longevity(object, class_col = NULL, posterior_threshold = 0.1, sub_class = FALSE)` in **new file** `R/longevity.R`.

### 4.2 Algorithm (posterior-weighted phase aggregation)

Inputs from the `sef_fit` object:

- `phase_prob[i, k]` — n × K posterior matrix.
- `class[i]` — categorical vector (default = `class_col` from the fit; overridable via the `class_col` argument; if `sub_class = TRUE`, build the cross factor `class:subclass` using `subclass_col` from the fit if available).
- `phase[i]` — the MAP (hard) phase assignment from `object$phase` (already provided by `fit_sef()`).
- `phase_envelope[k]` — for each phase `k`, the chronological envelope `[min(date_min) over i with phase[i] == k, max(date_max) over i with phase[i] == k]`. Computed once. Phases that contain no find with chronology have `phase_envelope[k] = c(NA, NA)` and are silently dropped from the active set `K_C` for any class.

Steps:

1. For each pair (class `C`, phase `k`): `weight[C, k] = mean(phase_prob[i, k]) over i with class[i] == C`.
2. Active phase set per class: `K_C = { k : weight[C, k] >= posterior_threshold }`.
3. Class longevity:
   - `longevity_min = min over k in K_C of phase_envelope[k].min`
   - `longevity_max = max over k in K_C of phase_envelope[k].max`
   - `longevity_span = longevity_max - longevity_min`
   - `dominant_phase = which.max(weight[C, ])`

### 4.3 Output schema

```r
data.frame(
  class            = factor,
  longevity_min    = numeric,        # signed year (BCE negative if dataset uses that convention)
  longevity_max    = numeric,
  longevity_span   = numeric,        # max - min
  dominant_phase   = integer,
  n_finds          = integer,
  weight_matrix    = list-column     # numeric vector of length K, the P(phase=k|class=C)
)
```

One row per class (or per `class:subclass` if `sub_class = TRUE`).

### 4.4 Edge cases

- Class with `K_C = empty set` (no phase passes the threshold): row emitted with `NA` longevity values; one cumulative `warning()` listing such classes.
- Class with a single find: row computed as usual; `n_finds = 1`.
- Threshold validation: `posterior_threshold` must be in `[0, 1]`; outside → `stop()`.

### 4.5 Plot companion

`gg_longevity(longevity_table, fit = NULL)` in `R/gg_plots.R`. A horizontal Gantt-style chart: one segment per class from `longevity_min` to `longevity_max`, ordered by `longevity_min` ascending, coloured by `dominant_phase` using the existing `.phase_colours` palette. Axis: chronology (with negative = BCE). Tooltip via `as_plotly()` shows `n_finds`, `weight_matrix`. Requires `ggplot2` (Suggests-gated like the other `gg_*` functions).

## 5. Feature 3 — rcarbon adapter

### 5.1 Function

`chronology_from_rcarbon(cal_dates, method = c("hpd", "median_iqr", "weighted_mean"), hpd = 0.95, ids = NULL, bce_negative = TRUE)` in **new file** `R/chronology.R`.

### 5.2 Input

- `cal_dates`: an object of class `CalDates` produced by `rcarbon::calibrate()`. Each element is a calibrated probability density on a calendar grid.
- `ids`: optional character vector of length `length(cal_dates$grids)`, used as the `id` column of the output. If `NULL`, defaults to `paste0("cal_", seq_along(...))`.

### 5.3 Reduction methods

- `"hpd"` (default): for each calibrated density, compute the Highest Posterior Density region at the requested level (`hpd`, default 0.95) using `rcarbon::hpdi()`. `date_min` = lowest endpoint, `date_max` = highest endpoint, `date_mid = (date_min + date_max) / 2`.
  - If the HPD region is **disjoint** (e.g. plateau in the calibration curve), still return the outer min/max but flag it: `attr(out$date_min, "disjoint_hpd")` set to a logical vector of length `n`. The Shiny app and `gg_longevity()` can choose to display the flag.
- `"median_iqr"`: weighted median = central; 25%–75% quantiles weighted by density form `[date_min, date_max]`.
- `"weighted_mean"`: weighted mean and SD; `date_min = mean - 2*sd`, `date_max = mean + 2*sd`.

### 5.4 Output schema

```r
data.frame(
  id        = character,
  date_min  = numeric,    # signed year, BCE negative if bce_negative = TRUE
  date_max  = numeric,
  date_mid  = numeric
)
```

Suitable for `merge()` with the user's main dataset prior to `fit_sef()`.

### 5.5 Sign convention

`bce_negative = TRUE` (default): `rcarbon` returns calibrated dates in calBP; we convert to calBCE/CE via `1950 - calBP` (calBCE will then be negative). With `bce_negative = FALSE`, the raw calBP is returned (rare use case).

### 5.6 Suggests

Add `rcarbon (>= 1.5.0)` to `DESCRIPTION` Suggests. Runtime gate:

```r
if (!requireNamespace("rcarbon", quietly = TRUE)) {
  stop("Package 'rcarbon' is required for chronology_from_rcarbon(). ",
       "Install with: install.packages('rcarbon')", call. = FALSE)
}
```
