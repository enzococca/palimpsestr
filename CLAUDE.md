## Project Overview

**palimpsestr** is an R package (v0.7.0) for probabilistic decomposition of archaeological palimpsests using Stratigraphic Entanglement Fields (SEF). It estimates latent depositional phases from spatial, stratigraphic, chronological, and cultural similarity data via diagonal Gaussian mixture EM.

## Build & Development Commands

```bash
# Check package (runs tests, builds docs, checks CRAN compliance)
R CMD check .

# Build source tarball
R CMD build .

# Install locally
R CMD INSTALL .

# Run tests only
Rscript -e 'testthat::test_dir("tests/testthat")'

# Regenerate documentation from roxygen2 comments
Rscript -e 'roxygen2::roxygenise()'
```

## Architecture

### Core Pipeline

The central workflow is: simulate/load data ‚Üí `fit_sef()` ‚Üí diagnostics/plots/export.

1. **`R/simulate.R`** ‚Äî `archaeo_sim()` generates synthetic palimpsest datasets with controllable mixing levels
2. **`R/utils.R`** ‚Äî Internal engine: `feature_matrix()` builds scaled feature vectors, `em_diag_gmm()` runs the diagonal Gaussian mixture EM, `softmax_negdist()` initializes probabilities, `build_context_penalty()` handles stratigraphic constraints
3. **`R/fit.R`** ‚Äî `fit_sef()` is the main entry point; returns an S3 `sef_fit` object containing phase assignments, probabilities, entropy, SEI matrix, energy, and model stats (loglik, BIC, PDI). Also contains `detect_intrusions()`, `pdi()`, `predict_phase()`, `as_phase_table()`
4. **`R/sei.R`** ‚Äî `sei_matrix()` computes the pairwise Stratigraphic Entanglement Index (O(n^2) double loop); `local_sei()` aggregates per-row
5. **`R/ese.R`** ‚Äî `ese()` computes Excavation Stratigraphic Energy (also O(n^2))
6. **`R/diagnostics.R`** ‚Äî `compare_k()` runs `fit_sef()` across multiple K values; `as_sf_phase()`/`as_sf_links()` export to sf for GIS; `phase_diagnostic_table()` builds combined output; `plot_energy()` base R plot

### Visualization Layer

- **`R/plot.R`** ‚Äî Base R plots: `plot_phasefield()`, `plot_entropy()`, `plot_sei_profile()`
- **`R/gg_plots.R`** ‚Äî ggplot2 equivalents: `gg_phasefield()`, `gg_entropy()`, `gg_energy()`, `gg_intrusions()`, `gg_compare_k()`, `gg_map()` (overlays on sf geometries). Uses colour-blind-safe palette (`.phase_colours`) and custom theme (`.theme_sef`)

### Data I/O

- **`R/db_connect.R`** ‚Äî `read_db()` loads from any DBI connection with column mapping; `load_geometries()` reads shapefiles/geopackages/databases via sf
- **`R/report.R`** ‚Äî `report_sef()` generates interpretive text reports in English or Italian
- **`R/data.R`** ‚Äî Dataset documentation for `demo_easy`, `demo_moderate`, `demo_compressed`
- **`data/`** ‚Äî Three `.rda` demo datasets with varying mixing levels (5%, 30%, 50%)

### Key S3 Class

`sef_fit` is the central object. All diagnostic, plotting, and export functions expect it. Key slots: `$data`, `$phase`, `$phase_prob`, `$entropy`, `$sei_matrix`, `$local_sei`, `$energy`, `$model_stats` (contains `$pdi`, `$loglik`, `$bic`).

## Dependencies

- **Core** (Imports): stats, utils, graphics, grDevices, methods ‚Äî no external compiled deps
- **Optional** (Suggests): ggplot2 + viridis + ggrepel for `gg_*` plots, sf for GIS export, DBI + RSQLite/RPostgres for database I/O, testthat/knitr/rmarkdown for dev

## Conventions

- All exported functions use roxygen2 `@export` with markdown-enabled docs (`Roxygen: list(markdown = TRUE)`)
- NAMESPACE is auto-generated by roxygen2 ‚Äî edit roxygen comments, not NAMESPACE directly
- Optional dependencies are checked at runtime with `requireNamespace(..., quietly = TRUE)`
- Internal helpers are prefixed with `.` (e.g., `.theme_sef`, `.phase_colours`, `.check_ggplot`)
- Input validation uses `check_required_columns()` from utils.R
- Tests use testthat edition 3 (`Config/testthat/edition: 3`)
